# Тестове завдання: QA Engineer (Taxer)

Даний репозиторій містить результати комплексного тестування (Manual + Automation) веб-додатку для завантаження та перегляду сертифікатів.

## 📁 Структура проекту

* [**bug-report.md**](./bug-report.md) — детальний звіт про 12 знайдених дефектів (Critical, High, Medium, Minor).
* [**bug-report.pdf**](./bug-report.pdf) — універсальна версія звіту для завантаження.
* **tests/** — автоматизовані E2E тести на базі Playwright (TypeScript).
* **pages/** — реалізація шаблону Page Object Model (POM).
* **report-crashing.png** — скріншот звіту, що фіксує краш інтерфейсу (Bug #1).
* **report-stable.png** — скріншот звіту при успішному рендерінгу після оновлення.

## 🚀 Як запустити автотести

Для запуску тестів у вашому середовищі виконайте наступні кроки:

1.  Встановіть залежності (перебуваючи в корені проекту):
    ```bash
    npm install
    ```
2.  Встановіть браузери Playwright (якщо не встановлені):
    ```bash
    npx playwright install
    ```
3.  Запустіть виконання тестів:
    ```bash
    npx playwright test
    ```
4.  Перегляньте детальний HTML-звіт:
    ```bash
    npx playwright show-report
    ```

## 💡 Особливості реалізації та аналіз

* **Валідація дефектів:** Тести спеціально налаштовані на перевірку бізнес-логіки. Тест на **Bug #2 (ПІБ)** стабільно падає, що є очікуваною поведінкою для підтвердження наявності дефекту.
* **Аналіз плаваючих помилок (Flaky Bugs):** Під час автоматизації **Bug #1 (LocalStorage)** було виявлено Race Condition. Додаток демонструє нестабільність при оновленні сторінки (F5). У проекті додано два варіанти скріншотів (`report-crashing` та `report-stable`) для демонстрації обох станів додатка.
* **Технологічний стек:** Playwright, TypeScript, Page Object Model. Реалізовано складну імітацію Drag-and-Drop подій через `DataTransfer` для тестування зони завантаження.
